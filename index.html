Introduction – Pourquoi les agents IA explosent en 2025/2026 ?

Niveau ★☆☆☆☆

Un agent IA moderne n’est plus juste un chatbot. C’est un système autonome qui :

  • raisonne étape par étape
  • utilise des outils (API, calculatrice, web, code…)
  • garde une mémoire longue
  • collabore avec d’autres agents
  • prend des décisions complexes sur plusieurs étapes

Les frameworks les plus utilisés aujourd’hui :

  • LangGraph (recommandé par LangChain pour la prod)
  • CrewAI (très simple pour multi-agents rôle-based)
  • AutoGen (Microsoft – conversations entre agents)
  • OpenAI Swarm / Agents (léger et géré)

Étape 1 – Premier agent ReAct ultra-basique

Niveau ★★☆☆☆

Le pattern ReAct = **Reason** (pense) + **Act** (agis) + observe le résultat → boucle jusqu’à la réponse finale.

from openai import OpenAI
client = OpenAI()

def obtenir_meteo(ville: str) -> str:
    return f"{ville} : 24°C, ensoleillé (simulation 2026)"

tools = [{
    "type": "function",
    "function": {
        "name": "obtenir_meteo",
        "description": "Récupère la météo d'une ville",
        "parameters": {
            "type": "object",
            "properties": {"ville": {"type": "string"}},
            "required": ["ville"]
        }
    }
}]

question = "Quel temps fait-il à Osasco aujourd'hui ?"
messages = [{"role": "user", "content": question}]

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    tools=tools
)

# ← Ici commence la vraie boucle ReAct (voir étape 2)
print(response.choices[0].message.tool_calls)

Étape 2 – Boucle complète + plusieurs outils

Niveau ★★☆☆☆ → ★★★☆☆

Ajoute une boucle while, plusieurs outils, gestion d’erreurs.

# Extrait simplifié de boucle ReAct
while True:
    resp = client.chat.completions.create(...)
    if resp.choices[0].message.tool_calls:
        for call in resp.choices[0].message.tool_calls:
            if call.function.name == "obtenir_meteo":
                args = json.loads(call.function.arguments)
                result = obtenir_meteo(**args)
                messages.append({"role": "tool", "content": result, "tool_call_id": call.id})
    else:
        final = resp.choices[0].message.content
        break

Étape 3 – Ajouter de la mémoire persistante

Niveau ★★★☆☆

Options simples → avancées :

  • Liste Python (history dans messages)

from langchain.memory import ConversationBufferMemory

Étape 4 – Contrôle moderne avec LangGraph

Niveau ★★★★☆

LangGraph remplace la boucle while par un **graphe d’état** → branches, cycles, human-in-the-loop, persistance, debug facile.

Exemple minimal :

from langgraph.graph import StateGraph, START, END

graph = StateGraph(...)
graph.add_node("agent", call_llm)
graph.add_node("tools", execute_tools)
graph.add_conditional_edges("agent", should_continue)
graph.add_edge("tools", "agent")
graph.add_edge(START, "agent")
app = graph.compile()
